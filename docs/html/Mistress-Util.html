NAME

Mistress::Util - Often-used basic functions for Mistress;

VERSION

version 0.001

SYNOPSIS

    use Mistress::Util qw{
        pcf pcf_e pcf_r pcf_w
        conf2pcf
        e r w
    };

    my $pcf = pcf($foo);      # get a Path::Class::File or dies
    my $pcf = pcf_e($foo);    # idem, but also dies unless $foo exists
    my $pcf = pcf_r($foo);    # idem, but also dies unless $foo is readable
    my $pcf = pcf_w($foo);    # idem, but also dies unless $foo is writable

    # Build a Path::Class::File by specifying a path after a configuration key
    my $stats_file = conf2pcf('Mistress/workdir', qw/ log stats.txt /);

    say "$pcf ", e($pcf) ? 'exists' : 'does not exist';
    say "$pcf is ", (r($pcf) ? '' : 'not '), 'readable';
    say "$pcf is ", (w($pcf) ? '' : 'not '), 'writable';

DESCRIPTION

This class exports (on request) short functions implementing things I find myself re-implementing again and again in other classes. Hopefully Mistress::Util will save me some keystrokes and make my sources more readable.

FUNCTIONS

pcf( $file ), pcf_e( $file ), pcf_r( $file ), pcf_w( $file )

Converts $file into a Path::Class::File if $file is a Path::Class::File object or a string that can be converted using Path::Class::file(); otherwise dies.

Additionally, pcf_e, pcf_r and pcf_w die if $file fails the corresponding -X test (i.e. -e, -r or -w).

conf2pcf( $key, @subpath )

Get the configuration value corresponding to $key, interpret it as a Path::Class::Dir and append @subpath to it to make a Path::Class::File which is returned.

Dies if $key as no associated configuration value (Mistress->config->get returns undef).

e( $any ), r( $any ), w( $any )

These functions do the same thing as -e, -r and -w but won't warn upon undef. Moreover, if their argument is an object that can stat (e.g. Path::Class), they will call that method before the -X call, which (hopefully) Does What You Want.
