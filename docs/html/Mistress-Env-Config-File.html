NAME

Mistress::Env::Config::File - The basic (and default) way to access the configuration.

VERSION

version 0.001

SYNOPSIS

    Mistress->config->load('config.yml');

    # later, somewhere else:

    my $opt = Mistress->config->get('some/option/key') // 'default';

DESCRIPTION

This class (possibly referred as "MECF" in other files) allows any other Mistress class to access the contents of the configuration file provided via the -c switch on command-line invocation.

This is the default config component loaded by Mistress. Therefore, all other Mistress core classes use the MECF specification style ("some/option/key") when querying the configuration. So, if you choose to use another config component, ensure its get method supports the MECF-style specification too!

METHODS

load( $path )

Update Mistress' configuration according to the file that corresponds to $path.

$path may be either a string or a Path::Class::File object. This method will die if the file doesn't exist or can't be read.

get( "CategoryA/.../KeyC" )

Give access to the configuration produced by YAML::XS from the file which path has been given to load and is stored in location.

get expects a string as its only argument. This string specifies which configuration key you want the value. Enumerate all successive keys separated with a slash (/), like in the following example:

    # Our configuration looks like this (YAML):
    x: 42
    A:
        a: 1
        b: 2

    # To get 42:
    Mistress->config->get('x');

    # To get 2:
    Mistress->config->get('A/b');

If you ask for a non-terminal value (like A in our previous example), you will get the hashref below A, but not the original one, a cloned hashref, so that configuration data never change (until you call load, of course). So, if you call get without arguments (or with an empty string), you'll get a copy of the whole configuration hashref.

If you specify an unknown key somewhere in your specification string, get returns undef and logs that (at notice level).
